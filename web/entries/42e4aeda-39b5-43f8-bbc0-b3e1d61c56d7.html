--- title: "The infamous Repository Pattern in PHP" comment: "El patrón de repositorio es una abstracción de una fuente de datos que se comporta como una colección que contiene objetos de dominio (valor). Este repositorio, a menudo presentado como una interfaz, se considera la única fuente confiable para recuperar y almacenar objetos o entidades de dominio. La pregunta de si combinar un repositorio y una fábrica en un solo servicio es algo que no puedo responder para tu situación, ya que personalmente prefiero mantener mis servicios separados para que puedan evolucionar de forma independiente. La capa de mapeo de relaciones de objetos (ORM) es una capa de abstracción sobre una base de datos (relacional). El objetivo del ORM es hacer que la interacción con la base de datos sea un enfoque orientado a objetos, actualizando las columnas como propiedades en la entidad y traduciendo de ida y vuelta con la base de datos." link: "https://doeken.org/blog/repository-pattern?ref=reddit" ---
